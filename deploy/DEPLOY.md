# 部署指南

## 1. OpenClaw部署

### 1.1 环境准备

- 安装Node.js 14.0+ 
- 安装OpenClaw CLI

### 1.2 安装步骤

1. **克隆仓库**
   ```bash
   git clone https://github.com/your-username/papermaking-skill.git
   cd papermaking-skill
   ```

2. **安装到OpenClaw**
   ```bash
   openclaw skills install ./skills/papermaking-mentor.skill
   ```

3. **验证安装**
   ```bash
   openclaw skills list
   ```

4. **启动OpenClaw服务**
   ```bash
   openclaw start
   ```

### 1.3 接入微信/企业微信/QQ

1. **微信接入**
   - 登录OpenClaw管理后台
   - 进入"渠道管理"→"微信"
   - 按照指引完成微信公众号或企业微信的配置

2. **QQ接入**
   - 登录OpenClaw管理后台
   - 进入"渠道管理"→"QQ"
   - 按照指引完成QQ机器人的配置

### 1.4 使用方法

- 在微信/企业微信/QQ中@机器人，直接提问
- 机器人会根据知识库内容提供专业回答

2. **导入知识库**
   - 进入"知识库"→"创建知识库"
   - 上传knowledge/目录下的所有.md文件

## 5. 常见问题及解决方案

### 5.1 OpenClaw部署问题

- **问题**：安装失败
  **解决方案**：检查Node.js版本，确保OpenClaw CLI已正确安装

- **问题**：机器人无响应
  **解决方案**：检查网络连接，确保OpenClaw服务正常运行

### 5.2 RAG平台部署问题

- **问题**：知识库导入失败
  **解决方案**：检查文件格式，确保文件大小符合平台要求

- **问题**：回答不准确
  **解决方案**：调整系统提示词，增加相关知识库内容

### 5.3 本地部署问题

- **问题**：依赖安装失败
  **解决方案**：使用国内镜像源，如pip install -i https://pypi.tuna.tsinghua.edu.cn/simple

- **问题**：应用启动失败
  **解决方案**：检查端口是否被占用，尝试使用其他端口

### 5.4 腾讯云部署问题

- **问题**：云函数超时
  **解决方案**：增加云函数的超时时间，优化代码性能

- **问题**：API网关访问失败
  **解决方案**：检查API配置，确保路径和方法正确
